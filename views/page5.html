Conclusion

  • add_boxRate your experience with the frameworks that you chose.
    For the backend, we chose Node.js with Express framework, we would rate it 10 out of 10. And we would rate 8/10 for Materialize, our choice for the frontend framework. It was fairly easy to understand and use, however, we found it had very limited options to work with.
  • add_boxDid they perform to your expectations?
    For the backend, we chose Node.js with Express framework. It performed just as our expectations. It was easy to install, configure, and use. With Materialize, the functions we used worked to our expectations, however, it was just lacking in variety.
  • add_boxWere they difficult to install or configure?
    It was easy to install and configure node.js and configure. Node.js has a package management system called NPM, we can initialize a project very easily by running command “npm init”. We can also install dependencies easily by running npm commands. For example, to install Express framework, we can run command “npm install express”. Materialize was also easy to install and configure, we had no issues with it.
  • add_boxWas it easy to create the page(s) with it?
    It was fairly easy to create pages with node.js. Initially, we were planning to host HTML files using express.static function. But to avoid redundancy in the HTML code, we decided to take advantage of Node.js’s rendering engines. In our app, we used EJS rendering engine, it has similar notations as HTML, with extra functionalities such as “include”. We used it to have one global navbar and footer, and we included it in each page, so to make changes, we can just edit the global code, and changes will take effect on all pages. We can say the same with Materialize, as it was extremely straightforward to follow and understand how to implement its features into our pages. It definitely offered its services with UI/UX experience first in mind.
  • add_boxWas the learning curve steep compared with regular HTML/CSS/Javascript/PHP?
    For backend, learning Node.js was as not as steep at first, because it is very different from PHP, and we didn’t know much about routing. However, the learning curve gets much steeper as we learned more, since it uses the same syntax as JavaScript, and NPM provides many packages that we can easily use in our app. We felt like it was not a steep learning curve with with Materialize, as it was very straightforward from the beginning. Tutorials and examples were to the point and very easy to follow and to manipulate for our own customization.
  • add_boxWould you have done things differently in retrospect?
    Yes, in retrospect, we would have designed the pages ahead of time to have a consistent theme. We made our pages individually, so they they were not as consistent. Therefore, when we added a global CSS styling, some pages were not compatible with it. With Materialize, it's 50% yes and no, since it was easy to use, but options in features and functions were limited.
  • add_boxDid you regret your choice of frameworks?
    We don’t regret our choice of frameworks.
